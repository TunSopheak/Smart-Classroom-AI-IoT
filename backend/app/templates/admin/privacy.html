Phase 12 Privacy Management

Storage & Privacy Policy

Explain how the system handles face data, attendance records, behavior events, and camera recordings.

Admin Storage
Product Principle: Smart classroom monitoring should be useful for teachers, but it must also be careful with privacy and consent.

Data Stored by System

  • Student identity and QR code information
  • Attendance records per class session
  • QR and FACE attendance event logs
  • AI behavior monitoring events
  • IoT sensor readings and automation logs
  • Camera recordings saved in local storage
  • Face dataset metadata for recognition training

Privacy Rules

  • Use volunteer/demo faces only during project demonstration
  • Do not publish raw classroom recordings without permission
  • Recordings are stored locally and are ignored by Git
  • Teachers/admins can delete recordings from the system
  • Event logs are for review and accountability
  • Future product version should include user roles and login protection

Recording Retention Policy

For a school product, recordings should not be kept forever. The admin should review and delete old recordings after they are no longer needed.

Manage Recordings System Health
Important: This project is a prototype. For real deployment, the school should define clear consent, access control, retention period, and data protection policy.
